# 專案憲法 - Claude 憲法原則應用

> 基於 Anthropic Claude 憲法精神，為瑪利歐賽車世界參數專案建立的行動指南

---

## 🎯 專案核心使命

**成為最值得信賴的瑪利歐賽車數據分析工具**，為玩家提供：

- 準確的遊戲數據分析
- 直觀的操作體驗
- 多語言的無障礙存取
- 安全可靠的數據處理

---

## ⚖️ 四大核心原則（優先順序）

### 1. 🛡️ 廣泛安全（最高優先級）

**絕不破壞用戶對專案的監督和信任**

#### 具體實施：

- **安全性**: 零容忍安全漏洞，定期執行 `pnpm audit`
- **隱私保護**: 所有數據本地儲存，不上傳用戶資訊
- **透明度**: 開源所有程式碼，公開所有依賴關係
- **穩定性**: 任何變更必須通過完整測試

#### 檢查清單：

```bash
# 每次提交前必須執行
pnpm audit --audit-level moderate
pnpm build
pnpm test  # 如果有測試
```

### 2. 🤝 廣泛道德

**成為一個善良、智慧、有品質的數位工具**

#### 具體實施：

- **誠實**: 數據來源透明，演算法邏輯公開
- **包容性**: 支援 5 種語言，考慮無障礙設計
- **謙遜**: 承認資料限制，鼓勵用戶回饋
- **責任**: 對用戶時間和數據負責

#### 硬性約束：

- ❌ 絕不收集個人資訊
- ❌ 絕不強制推薦特定組合
- ✅ 必須標明資料來源和更新時間
- ✅ 必須提供明確的數據說明

### 3. 📋 遵循專案指南

**符合已建立的技術規範和最佳實踐**

#### 具體實施：

- **Git Workflow**: 嚴格遵循 `.opencode/skills/git-workflow`
- **程式碼標準**: 遵循 `code-standards` 和 `react-best-practices`
- **國際化**: 遵循 `i18n-workflow` 規範
- **文檔同步**: 遵循 `readme-maintenance` 指南

#### 衝突處理：

當指南與實際需求衝突時：

1. 優先考慮用戶體驗
2. 在不犧牲安全性的前提下靈活調整
3. 記錄決策原因供未來參考

### 4. 💡 真正有幫助

**為用戶提供實質性價值和優質體驗**

#### 具體實施：

- **準確性**: 確保所有瑪利歐賽車數據準確無誤
- **易用性**: 直觀的界面設計，清晰的操作指引
- **效能**: 快速載入，流暢的互動體驗
- **持續改進**: 根據用戶回饋持續優化

#### 價值判斷啟發式：

- 這個功能對玩家真的有幫助嗎？
- 是否簡化了複雜的決策過程？
- 是否提供了獨特的價值？

---

## 🔄 衝突解決機制

### 價值權衡框架

當原則之間發生衝突時，按以下順序優先：

1. **安全 > 道德**: 絕不為了功能性犧牲安全性
2. **道德 > 指南**: 當規範阻礙用戶福祉時，優先道德
3. **指南 > 幫助性**: 在安全道德前提下，遵循技術規範
4. **幫助性**: 作為最後的考量因素

### 實際案例

#### 案例 1：安全性 vs 功能性

- **情境**: 新功能需要引入有漏洞的依賴
- **決策**: 放棄該功能，尋找更安全的實現方式

#### 案例 2：道德 vs 效能

- **情境**: 優化會降低多語言支援的品質
- **決策**: 優先保持語言支援的完整性

#### 案例 3：指南 vs 用戶需求

- **情境**: 用戶需要不符合現有程式碼標準的功能
- **決策**: 在不影響安全性的前提下，靈活調整標準
